# 📚 Project 1 — Customer Support Knowledge Base

A structured internal knowledge base built to simulate the documentation system I maintained at TTEC Financial Services. This is the kind of resource that reduces repeat tickets, speeds up onboarding, and empowers agents to resolve issues without escalation.

---

## Structure

| Folder | Contents |
|--------|----------|
| [`/troubleshooting`](./troubleshooting/) | Step-by-step guides for the most common technical issues |
| [`/escalation`](./escalation/) | Escalation decision trees and handoff templates |
| [`/faq`](./faq/) | Customer-facing FAQ for common account and access questions |
| [`/sop`](./sop/) | Standard Operating Procedures for daily support workflows |

---

## Why This Matters

In my previous role, I maintained and updated the internal knowledge base after every resolved ticket. The result: fewer repeat issues, faster resolution for new agents, and a team that didn't have to reinvent the wheel every time a familiar problem appeared.

This documentation was a direct contributor to our team's sustained 96–100% QA scores.
